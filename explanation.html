Interactive Demo — Block Size 3

Imagine 7 pixel bits from an image — the switches below. Each switch is labeled with a binary number from 001 to 111. Flip a switch and watch which bulbs light up. Notice anything? The switch label tells you exactly which bulbs it affects — switch 011 flips bulbs 1 and 2, switch 101 flips bulbs 1 and 3. The label is the XOR mask.

Now here's the trick: given any starting switch state, you can always reach any target bulb pattern by flipping at most one switch. That one flip is how 3 secret bits get hidden inside 7 cover bits — changing just one pixel.

How It Works

Each switch is labeled with a binary number (001–111). Flipping a switch XORs those bits into the bulb state. To encode a secret, the algorithm finds a single switch to flip that produces exactly the target bulb pattern — changing at most one bit in the cover image per block.

  • Switches represent LSBs of image pixels (the cover data)
  • Bulbs represent the secret message bits being hidden
  • Switch "011" toggles both bulb 1 and bulb 2 — its label is the XOR mask
  • To encode secret bits, find the switch whose label equals current_bulbs XOR target_bulbs
  • If no flip is needed (already correct), no pixel is changed at all
  • To decode, the receiver computes the XOR of all on switch labels — that's the secret
